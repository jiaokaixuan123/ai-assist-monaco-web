📊 修复说明

问题原因：

Monaco Editor 默认不包含 Python 的词法分析器（Monarch tokenizer），导致基础语法高亮失效。只有语义高亮（需要 LSP 服务器）在工作，但覆盖不全面。

解决方案：

  1. 创建了完整的 Python Monarch tokenizer 配置文件
  2. 注册了 Python 语言配置（括号匹配、注释、折叠等）
  3. 扩展了主题规则，包含所有 Python token 类型
  4. 确保词法高亮（Monarch）+ 语义高亮（LSP）协同工作

技术实现：

  • pythonLanguage.ts - Python Monarch tokenizer 定义
  • App.tsx - 注册语言和应用主题
  • global.css - CSS 样式覆盖（确保颜色生效）
💡 提示

如果某些符号仍然没有颜色或颜色不正确：

  • 检查浏览器控制台是否有错误
  • 确认 Pyright LSP 服务器是否正常连接
  • 尝试刷新页面（Ctrl+F5 强制刷新）
  • 检查"语言服务"和"语义高亮"开关是否开启
