🐛 装饰器语法高亮问题调试

问题描述

当代码中出现 @decorator 时，后续的函数或类定义整行变成黄色或青色。

@property
def get_value(self):  ← 这一行全部变黄色
    return self._value

诊断步骤

  1. 打开浏览器开发者工具（F12）
  2. 切换到 Console 标签
  3. 粘贴以下代码并执行：
// 检查装饰器
const editor = monaco.editor.getModels()[0]
const text = editor.getValue()
const lines = text.split('\n')

// 找到包含装饰器的行
lines.forEach((line, idx) => {
    if (line.trim().startsWith('@')) {
        console.log(`行 ${idx + 1}: ${line}`)
        
        // 检查这一行的 tokens
        const tokens = monaco.editor.tokenize(line, 'python')
        console.log('Tokens:', tokens)
    }
})

检查 Semantic Tokens

// 检查 LSP semantic tokens
if (window.pyrightSemanticDebug) {
    console.log('Semantic Legend:', window.pyrightSemanticDebug.semanticLegend)
}

// 检查装饰器相关的 decorations
const decorations = editor.getAllDecorations()
    .filter(d => d.options.inlineClassName)
    
console.log('Decorations count:', decorations.length)
decorations.forEach(d => {
    console.log({
        range: `${d.range.startLineNumber}:${d.range.startColumn}-${d.range.endLineNumber}:${d.range.endColumn}`,
        class: d.options.inlineClassName,
        text: editor.getValueInRange(d.range)
    })
})

可能的原因

  • LSP Semantic Tokens
    • Pyright 可能将装饰器后的整个函数/类定义识别为一个语义 token
    • Semantic token 的范围过大，覆盖了多行
  • CSS 样式继承
    • 装饰器的黄色样式可能意外继承到后续元素
    • CSS 选择器过于宽泛
  • Monaco 渲染问题
    • Monaco 可能在渲染 semantic tokens 时出错
    • Token 边界计算错误

临时解决方案

// 在浏览器控制台执行，临时禁用语义高亮
// 查看是否还有颜色覆盖问题

// 方法 1: 通过设置面板关闭"语义高亮"

// 方法 2: 在控制台手动清除所有装饰器
const editor = monaco.editor.getModels()[0]
const decorations = editor.getAllDecorations()
    .filter(d => d.options.inlineClassName)
    .map(d => d.id)
    
editor.deltaDecorations(decorations, [])
